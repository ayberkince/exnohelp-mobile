# Client Request Creation Flow v1

## Purpose
This document defines how a client creates a support request.

The goal is to make request creation:
- fast
- clear
- safe
- structured enough for helpers to evaluate
- not emotionally overwhelming

---

## Core Goal
A client should be able to create a request in a few minutes and feel confident that helpers will understand what is needed.

---

## Request Creation Steps

### Step 1 — Choose Service Category
Available categories:
- accompaniment to appointment
- waiting support
- return-home accompaniment
- paperwork support
- emotional companionship
- light practical help after appointment

### UX goal
Start with a simple, category-first choice.

---

## Step 2 — Date and Time
Client selects:
- date
- start time
- end time or duration

### UX goal
Make timing clear and structured for matching.

---

## Step 3 — Location
Client enters:
- city
- district
- meeting point
- optional end location

### UX goal
Location must be specific enough for helpers to evaluate feasibility.

---

## Step 4 — Support Details
Client adds:
- short title
- short description
- language preference
- optional gender preference
- mobility notes
- optional accessibility notes

### UX goal
The details should be practical, not medically detailed.

---

## Step 5 — Review Request
Client sees summary:
- category
- date/time
- location
- key notes
- language preference
- estimated price logic if relevant

### Trust notice
Helpers provide non-medical support only.

---

## Step 6 — Publish Request
Client publishes request to marketplace.

### Result
Request enters `open` status and becomes visible to eligible helpers.

---

## Request Creation Principles
- Keep the flow under 6 steps
- Ask only what helps matching and safety
- Avoid heavy medical questions
- Reinforce non-medical boundaries
- Make the final request summary easy to understand
