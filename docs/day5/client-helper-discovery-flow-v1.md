# Client Helper Discovery Flow v1

## Purpose
This document defines how a client browses, evaluates, and selects helpers.

The helper discovery experience must emphasize:
- trust
- clarity
- fit
- calm decision-making

---

## Core Goal
A client should be able to compare helpers confidently and understand why one helper may be a better fit than another.

---

## Discovery Entry Points
A client can enter helper discovery from:
- client home screen
- after onboarding
- after request creation
- from search tab
- from rebooking flow later

---

## Helper Card Requirements
Each helper card must show:
- profile photo
- first name
- languages
- district / area
- rating
- completed bookings count
- verification badge
- support categories
- hourly rate
- availability indicator

### UX goal
The helper card must support trust-first comparison at a glance.

---

## Helper Profile Requirements
The helper profile page must show:
- hero profile section
- trust badges
- rating and review count
- languages
- categories offered
- bio
- location area
- availability preview
- reviews
- clear platform safety note
- CTA to book

### UX goal
The profile should feel human, reassuring, and structured.

---

## Decision Support Rules
Clients should be able to compare helpers based on:
- language fit
- location fit
- price
- verification level
- completed bookings
- reviews
- support category fit

---

## Primary CTA
Book this helper

## Secondary CTA
Message may be added later only if booking context allows, but v1 should prioritize the booking flow.
