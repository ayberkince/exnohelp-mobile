# DB Auth and Role Model v1
## User Model
- id, clerkId, email
- role (CLIENT, HELPER, ADMIN)
- onboardingStatus (NOT_STARTED, IN_PROGRESS, COMPLETE)
- accountStatus (ACTIVE, SUSPENDED)
## Profiles
- ClientProfile: Emergency Contact, District, Preferences.
- HelperProfile: Bio, Hourly Rate, Categories, Verification Status.
