# Verification Ops v1

## Purpose
This document defines how helper verification is reviewed operationally.

---

## Verification Review Inputs
Operator should review:
- helper profile completeness
- ID verification submission
- phone/email completion
- payout readiness
- optional background document if used later

---

## Verification Outcomes
- approved
- rejected
- needs_action

---

## Needs Action Examples
- missing profile photo
- missing ID image
- unreadable document
- payout setup incomplete

---

## Rejection Examples
- identity mismatch
- suspicious submission
- clearly false information
- prohibited or unusable documentation

---

## Logging Rule
Every verification decision must record:
- reviewer
- result
- timestamp
- short internal note
