# Moderation Workflow v1

## Purpose
This document defines how platform admins/operators review reports and take action.

---

## Core Principle
Moderation decisions should be:
- documented
- proportional
- consistent
- linked to platform policy

---

## Moderation Flow
1. report is submitted
2. report enters open state
3. admin/operator reviews context
4. evidence is examined
5. severity is assessed
6. action is chosen
7. action is logged
8. report is closed or monitored further

---

## Severity Levels

### Low Severity
Examples:
- isolated rude message
- minor misunderstanding
- low-impact complaint

Possible actions:
- note only
- reminder
- warning

### Medium Severity
Examples:
- repeated lateness
- repeated cancellations
- pressure for off-platform behavior
- stronger boundary concerns

Possible actions:
- warning
- temporary restriction
- payout hold if relevant
- profile pause
- manual review state

### High Severity
Examples:
- harassment
- strong boundary violation
- identity deception
- repeated unsafe conduct
- serious report with evidence

Possible actions:
- immediate suspension
- booking cancellation
- refund review
- permanent removal after review

---

## Moderation Record Requirements
Every admin action should record:
- who reviewed
- what was reviewed
- what policy category applied
- what action was taken
- when it happened
- notes

---

## Fairness Rule
Not every report means guilt.
But repeated patterns should matter.

Moderation should evaluate:
- current case
- past reliability pattern
- prior warnings
- severity of conduct
