# No-Show and Reliability v1

## Purpose
This document defines how reliability issues are tracked and handled.

Reliability is a trust signal on this platform.

---

## Reliability Problems
Relevant reliability issues include:
- no-show
- repeated late arrival
- repeated last-minute cancellation
- failure to respond after accepting
- repeated incomplete bookings caused by user behavior

---

## No-Show Definition
A no-show occurs when a booked user fails to appear or meaningfully participate without acceptable communication.

Possible no-show parties:
- helper
- client

---

## No-Show Effects
### If helper no-shows
Possible effects:
- client refund review
- reliability score impact
- warning or restriction
- suspension review if repeated

### If client no-shows
Possible effects:
- refund reduction or no refund according to policy
- reliability score impact
- warning if repeated

---

## Reliability Tracking
Track at minimum:
- completed bookings
- cancellations
- no-shows
- lateness reports
- dispute frequency

These metrics should feed:
- trust indicators
- admin review
- enforcement decisions
