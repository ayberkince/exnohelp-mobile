# Reporting System v1

## Purpose
This document defines how users report problems on the platform.

The reporting system must be:
- easy to find
- easy to use
- structured enough for action
- serious enough to support moderation

---

## What users can report
A user should be able to report:

### User-related issues
- unsafe behavior
- harassment
- discrimination
- inappropriate conduct
- identity concerns

### Booking-related issues
- no-show
- severe lateness
- boundary violation
- unsafe support behavior
- misleading booking behavior

### Message-related issues
- harassment in chat
- inappropriate messages
- pressure for off-platform communication
- pressure for off-platform payment

### Payment-related issues
- suspected fraud
- payment manipulation
- misleading compensation behavior

---

## Report Types
Recommended report categories:
- no_show
- severe_lateness
- unsafe_behavior
- harassment
- discrimination
- prohibited_service_behavior
- off_platform_payment_request
- fraud
- suspicious_identity
- inappropriate_message
- other

---

## Report Entry Points
Users should be able to create reports from:
- user profile
- booking detail page
- conversation/chat
- support center

---

## Report Submission Fields
A report should collect:
- report type
- short description
- optional longer description
- optional screenshot or evidence later
- booking reference if applicable

---

## Reporting UX Rules
- reporting must be visible but not intrusive
- user should know report was submitted
- do not promise immediate action
- do promise review
- show support path for urgent concerns

---

## Report States
- open
- under_review
- action_taken
- dismissed
- closed
