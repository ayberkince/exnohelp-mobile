# Suspension and Enforcement v1

## Purpose
This document defines the enforcement ladder for problematic user behavior.

---

## Enforcement Levels

### Level 1 — Informational Reminder
Used for:
- minor first-time issues
- misunderstanding of boundaries

Effect:
- policy reminder only

---

### Level 2 — Warning
Used for:
- repeated minor misconduct
- moderate reliability issues
- first meaningful boundary issue

Effect:
- formal warning logged
- future behavior monitored more closely

---

### Level 3 — Temporary Restriction / Pause
Used for:
- repeated cancellations
- repeated no-show patterns
- moderate unsafe behavior
- verification concerns
- active review situations

Effect:
- limited platform actions
- temporary inability to accept or create bookings in some cases

---

### Level 4 — Suspension
Used for:
- serious harassment
- repeated misconduct
- fraud concerns
- severe boundary violations
- strong trust threat

Effect:
- access to normal product blocked
- review-state screen shown
- admin review required

---

### Level 5 — Ban
Used for:
- extreme misconduct
- severe fraud
- sustained unsafe behavior
- behavior fundamentally incompatible with platform trust

Effect:
- permanent removal according to policy

---

## Reliability-Based Enforcement
Reliability problems should also feed enforcement logic.

Relevant metrics:
- no-show count
- late cancellation count
- repeated lateness
- repeated failed responses

These should not instantly ban users, but repeated patterns should trigger review.
