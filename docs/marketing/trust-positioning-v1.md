# Trust Positioning v1

## Purpose
This document defines how trust and safety should be explained publicly.

---

## Core Public Trust Message
This platform is built around clear boundaries, safer coordination, and visible trust markers.

---

## Public Trust Points
The public website should clearly communicate:
- helpers provide non-medical support only
- profiles include trust-related information
- in-platform booking and payment is encouraged
- reporting tools exist
- moderation exists
- verification exists in the helper flow

---

## Important Trust Rule
Do not overpromise with language like:
- fully guaranteed safety
- medically supervised support
- professional clinical oversight

Trust should be communicated honestly:
- structured
- moderated
- clearer than informal arrangements
- designed to support safer experiences
