# Helper Onboarding Flow v1

## Purpose
This document defines how a new helper becomes ready to participate in the marketplace.

The onboarding must balance:
- speed
- trust
- clarity
- profile quality
- service-boundary discipline

---

## Core Onboarding Goal
A helper should finish onboarding understanding:
- what the platform is
- what they are allowed to do
- what they are not allowed to do
- how they will earn
- what is needed to become trusted and bookable

---

## Step 1 — Welcome / Role Confirmation
### Goal
Confirm that the user is joining as a helper.

### Screen content
- headline: Help people and earn flexible income
- explanation of non-medical support role
- reminder that this is not clinical work
- CTA: Continue as Helper

---

## Step 2 — Basic Identity Setup
### Required fields
- first name
- last name
- profile photo
- preferred language
- city
- district

### Optional
- additional languages

### UX goal
This should begin building trust immediately.

---

## Step 3 — Public Helper Profile
### Required fields
- short bio
- languages
- service categories
- hourly rate
- transport mode
- city/district
- availability basics

### Optional
- short experience statement
- why you want to help
- comfort preferences

### UX goal
Build a profile that feels human, calm, and trustworthy.

---

## Step 4 — Availability
### Required fields
- days available
- time ranges
- whether currently accepting requests

### UX goal
Create real marketplace readiness.

---

## Step 5 — Rules and Boundaries
### Must clearly explain
- this is non-medical support only
- no diagnosis
- no medication administration
- no nursing
- no emergency transport
- no clinical claims

### Required acceptance
- helper rules
- prohibited task policy
- platform behavior standards

---

## Step 6 — Verification Introduction
### Explain
To build trust, helpers are asked to complete verification steps.

### Show verification categories
- email
- phone
- ID
- payout setup
- additional document placeholder if required later

### UX goal
Make verification feel like a trust-building step, not punishment.

---

## Step 7 — First Action Prompt
After onboarding completion, helper should be guided to:
- Browse open requests
- Complete verification
- Improve profile quality

### Preferred first CTA
Complete verification

### Secondary CTA
Browse open requests

---

## Completion Condition
Helper onboarding is complete when:
- required profile fields are completed
- required categories and availability are set
- required rule acknowledgements are accepted

Verification approval is separate from onboarding completion.
