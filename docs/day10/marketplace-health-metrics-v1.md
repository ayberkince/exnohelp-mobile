# Marketplace Health Metrics v1

## Purpose
This document defines the key metrics that show whether the marketplace is healthy.

---

## 1. Liquidity Metrics
- number of open requests
- number of active helpers
- request-to-application rate
- request-to-booking conversion rate
- average time to first application
- average time to confirmed booking

---

## 2. Reliability Metrics
- booking completion rate
- helper cancellation rate
- client cancellation rate
- no-show rate
- severe lateness rate

---

## 3. Trust and Safety Metrics
- report rate per booking
- dispute rate per booking
- payout hold rate
- verification approval rate
- suspension rate

---

## 4. Financial Metrics
- gross booking value
- platform fee revenue
- average booking value
- refund rate
- payout volume

---

## 5. Operational Metrics
- pending verification count
- average verification review time
- open support count
- average report review time
- refund review backlog

---

## 6. Early Warning Metrics
These should be watched closely:
- rising no-show rate
- rising refund rate
- low request application rate
- low helper activation rate
- rising support burden per booking
