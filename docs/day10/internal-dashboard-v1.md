# Internal Dashboard v1

## Purpose
This document defines the internal KPI dashboard for founders and operators.

---

## Dashboard Sections

### 1. Marketplace Overview
- active clients
- active helpers
- open requests
- confirmed bookings
- completed bookings

### 2. Funnel Overview
- signup -> onboarding
- onboarding -> first action
- first action -> booking
- booking -> completion

### 3. Trust and Safety
- reports opened today/week
- disputes opened today/week
- no-show rate
- suspension count
- payout holds

### 4. Financials
- gross booking value
- platform revenue
- refunds
- pending payouts
- completed payouts

### 5. Ops Health
- pending verifications
- average verification review time
- support backlog
- report backlog

### 6. Retention
- repeat booking rate
- returning helpers
- WAU / MAU style activity metrics where useful
