# Analytics Framework v1

## Purpose
This document defines the analytics philosophy for the first version of the marketplace.

The analytics system exists to help the team understand:
- user behavior
- conversion bottlenecks
- trust and safety issues
- liquidity problems
- retention
- business performance

---

## Core Principles

### 1. Decision-Oriented Analytics
Every tracked metric should help answer a real product or business question.

### 2. Funnel Visibility
The platform must track where users drop in:
- signup
- onboarding
- request creation
- booking
- payment
- completion

### 3. Marketplace Visibility
The platform must measure both sides of the marketplace:
- client activity
- helper activity

### 4. Trust and Safety Visibility
The platform must measure:
- reports
- no-shows
- cancellations
- disputes
- verification progress

### 5. Operational Visibility
The platform must support manual ops with:
- queue metrics
- review times
- payout holds
- support load

---

## Measurement Categories
- acquisition
- activation
- conversion
- retention
- trust and safety
- marketplace liquidity
- financial performance
- support and operations
