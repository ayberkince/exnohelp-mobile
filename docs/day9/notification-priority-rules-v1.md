# Notification Priority Rules v1

## Purpose
This document defines which notifications are high, medium, or low priority.

---

## High Priority
These should usually trigger both in-app and email:
- booking confirmed
- booking canceled
- payment failed
- account under review
- verification approved if access changes
- payout held if it affects earnings
- serious support or trust update

---

## Medium Priority
These may trigger in-app and sometimes email:
- new request available for helper
- new application on request
- booking reminder
- payout completed
- refund completed

---

## Low Priority
These are typically in-app first and sometimes email later:
- complete your profile
- finish verification
- leave a review
- return to browse requests
- repeat booking prompt

---

## Priority Rule
Not every event needs email.
Use email for events that matter outside the session.
