# Lifecycle Messaging v1

## Purpose
This document defines messages tied to user progress and account state.

The purpose is to guide users from signup to meaningful usage.

---

## 1. Client Lifecycle Messages

### Stage A — Signed up, not onboarded
Message:
Complete your profile to start booking support.

### Stage B — Onboarded, no request yet
Message:
Create your first support request or browse helpers.

### Stage C — Request created, no booking yet
Message:
Your request is live. Review responses or browse helpers directly.

### Stage D — Booking completed
Message:
How did it go? Leave a review and book again anytime.

---

## 2. Helper Lifecycle Messages

### Stage A — Signed up, not onboarded
Message:
Complete your profile to start helping people and earning.

### Stage B — Onboarded, not verified
Message:
Finish verification to build trust and unlock more booking access.

### Stage C — Verified, no accepted booking yet
Message:
Browse open requests and apply to your first booking.

### Stage D — First booking completed
Message:
Great work. Keep your profile active to find more requests.

---

- verification needs action
