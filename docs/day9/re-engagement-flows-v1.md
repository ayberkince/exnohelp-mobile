# Re-engagement Flows v1

## Purpose
This document defines how inactive users are encouraged to return.

Re-engagement should be helpful, not spammy.

---

## 1. Client Re-Engagement
Trigger examples:
- signed up but never created request
- created request but never booked
- completed one booking and became inactive

Recommended messages:
- Need support again? Book trusted non-medical help when you need it.
- Your profile is ready. Create a request in just a few minutes.
- Browse helpers available in your area.

---

## 2. Helper Re-Engagement
Trigger examples:
- signed up but never completed onboarding
- onboarding complete but never verified
- verified but inactive
- previously active but no recent engagement

Recommended messages:
- Complete your profile to start receiving requests.
- Finish verification to build trust with clients.
- New support requests may be available in your area.

---

## 3. Re-Engagement Rules
- do not over-message
- only send when there is a meaningful next action
- tie the message to actual user state
